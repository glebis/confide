# CONFIDE — tools & dependencies

Every external tool CONFIDE uses, what it does, its role here, and where to find it. CONFIDE
is **local-first**: the default de-identification stack runs entirely on your machine; cloud
LLMs are used **only** for benchmark experiments on **synthetic** data, never on real sessions.

## De-identification stack (local)

| Tool | What it is | Role in CONFIDE | Link | License |
|---|---|---|---|---|
| **Natasha** | Russian NLP toolkit (NER, morphology) | Russian named-entity layer — names, locations, orgs | [github.com/natasha/natasha](https://github.com/natasha/natasha) | MIT |
| **scrubadub** | Python PII detection/cleaning | Deterministic email/URL detection in the regex layer | [github.com/LeapBeyond/scrubadub](https://github.com/LeapBeyond/scrubadub) | MIT/Apache |
| **phonenumbers** (libphonenumber) | Phone-number parsing | Robust phone detection in the regex layer | [github.com/daviddrysdale/python-phonenumbers](https://github.com/daviddrysdale/python-phonenumbers) | Apache-2.0 |
| **Ollama** | Local LLM runtime (Metal/CUDA) | Runs the local quasi-PII LLM layer (medications, ages, professions, contextual IDs) | [ollama.com](https://ollama.com) · [github.com/ollama/ollama](https://github.com/ollama/ollama) | MIT |
| **llama.cpp** | C/C++ LLM inference | Alternative local LLM engine (memory-efficient, reproducible) | [github.com/ggml-org/llama.cpp](https://github.com/ggml-org/llama.cpp) | MIT |
| **Qwen** (qwen2.5 / qwen3) | Open LLM family (Alibaba) | The local LLM used for quasi-PII recovery; the de-id default is `qwen2.5:3b` | [github.com/QwenLM/Qwen2.5](https://github.com/QwenLM/Qwen2.5) · [qwenlm.github.io](https://qwenlm.github.io) | Apache-2.0 |

## Report rendering

The Tufte-style HTML report (`results/benchmark-report.html`) uses **[Chart.js](https://www.chartjs.org)**
(MIT) for charts and the **EB Garamond** / **Monaspace** fonts. Generated by
`src/confide_eval/report/make_tufte_report.py`.

---

*Tool versions are pinned in `requirements.lock` (see [`REPRODUCIBILITY.md`](REPRODUCIBILITY.md)); the local LLM model id +
date are recorded per run in the registry (`caches/runs/`). Cloud providers expose no model
digest, so we record the served `id` string + date instead.*
